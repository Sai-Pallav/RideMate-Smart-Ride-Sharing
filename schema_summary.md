# Database Schema Summary

This document summarizes the database schema implemented for the Smart Community Ride Sharing Platform. All tables, columns, indexes, and constraints map 1:1 with `/ride-sharing/07-database-design.md`, utilizing native MySQL spatial geometry columns for geographic coordinate data.

---

## 1. Tables and Field Specifications

### 1. `users` (Core Auth Identity Anchor)
- `user_id` (PK, BIGINT, Auto-Increment)
- `phone_number` (UNIQUE, VARCHAR(50), NOT NULL)
- `email` (UNIQUE, VARCHAR(255), NOT NULL)
- `password_hash` (VARCHAR(255), Nullable for OTP-only auth)
- `phone_verified` (BOOLEAN, Default: `false`)
- `email_verified` (BOOLEAN, Default: `false`)
- `account_status` (ENUM: `'active', 'suspended', 'banned', 'deactivated'`, Default: `'active'`)
- `created_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP`)
- `updated_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP`)

### 2. `profiles` (User Profiles, 1:1 with Users)
- `profile_id` (PK, BIGINT, Auto-Increment)
- `user_id` (FK → `users.user_id`, CASCADE updates/deletes)
- `full_name` (VARCHAR(255), NOT NULL)
- `photo_url` (VARCHAR(2048), Nullable)
- `gender` (ENUM: `'male', 'female', 'other', 'prefer_not_to_say'`, Nullable)
- `institution_name` (VARCHAR(255), Nullable)
- `institution_domain` (VARCHAR(255), Nullable)
- `bio` (TEXT, Nullable)
- `created_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP`)
- `updated_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP`)

### 3. `vehicles` (Vehicles registered by Drivers)
- `vehicle_id` (PK, BIGINT, Auto-Increment)
- `user_id` (FK → `users.user_id`, CASCADE updates/deletes)
- `vehicle_type` (ENUM: `'two_wheeler', 'car'`, NOT NULL)
- `make` (VARCHAR(100), NOT NULL)
- `model` (VARCHAR(100), NOT NULL)
- `registration_number` (UNIQUE, VARCHAR(50), NOT NULL)
- `color` (VARCHAR(50), NOT NULL)
- `is_active` (BOOLEAN, Default: `true`)
- `created_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP`)

### 4. `rides` (Published Ride Offers)
- `ride_id` (PK, BIGINT, Auto-Increment)
- `driver_id` (FK → `users.user_id`, CASCADE updates/deletes)
- `vehicle_id` (FK → `vehicles.vehicle_id`, RESTRICT deletes)
- `source_label` (VARCHAR(255), NOT NULL)
- `source_location` (POINT (SRID 4326), NOT NULL)
- `destination_label` (VARCHAR(255), NOT NULL)
- `destination_location` (POINT (SRID 4326), NOT NULL)
- `ride_date` (DATEONLY, NOT NULL)
- `departure_time` (TIME, NOT NULL)
- `total_seats` (INT, NOT NULL)
- `available_seats` (INT, NOT NULL)
- `estimated_distance_km` (DECIMAL(10, 2), NOT NULL)
- `estimated_total_cost` (DECIMAL(10, 2), NOT NULL)
- `status` (ENUM: `'scheduled', 'ongoing', 'completed', 'cancelled'`, Default: `'scheduled'`)
- `recurring_template_id` (FK → `rides.ride_id` self-reference, SET NULL deletes)
- `preferences` (JSON, Nullable)
- `created_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP`)
- `updated_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP`)

### 5. `ride_routes` (Geo polyline route, 1:1 with Rides)
- `route_id` (PK, BIGINT, Auto-Increment)
- `ride_id` (FK → `rides.ride_id`, UNIQUE, CASCADE updates/deletes)
- `polyline_data` (JSON, NOT NULL)
- `total_distance_km` (DECIMAL(10, 2), NOT NULL)
- `created_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP`)

### 6. `ride_stops` (Intermediate ordered waypoints)
- `stop_id` (PK, BIGINT, Auto-Increment)
- `ride_id` (FK → `rides.ride_id`, CASCADE updates/deletes)
- `sequence_order` (INT, NOT NULL)
- `stop_label` (VARCHAR(255), NOT NULL)
- `stop_location` (POINT (SRID 4326), NOT NULL)
- `distance_from_source_km` (DECIMAL(10, 2), NOT NULL)
- `created_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP`)
- **Constraints**: Composite unique key on `(ride_id, sequence_order)`.

### 7. `bookings` (Passenger requests/joins, N:1 with Rides/Users)
- `booking_id` (PK, BIGINT, Auto-Increment)
- `ride_id` (FK → `rides.ride_id`, CASCADE updates/deletes)
- `passenger_id` (FK → `users.user_id`, CASCADE updates/deletes)
- `pickup_label` (VARCHAR(255), NOT NULL)
- `pickup_location` (POINT (SRID 4326), NOT NULL)
- `drop_label` (VARCHAR(255), NOT NULL)
- `drop_location` (POINT (SRID 4326), NOT NULL)
- `distance_traveled_km` (DECIMAL(10, 2), NOT NULL)
- `calculated_cost_share` (DECIMAL(10, 2), NOT NULL)
- `booking_status` (ENUM: `'pending', 'confirmed', 'declined', 'expired', 'cancelled', 'completed', 'no_show'`, Default: `'pending'`)
- `match_scenario` (ENUM: `'exact', 'partial_exit', 'partial_pickup'`, NOT NULL)
- `requested_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP`)
- `confirmed_at` (TIMESTAMP, Nullable)
- `completed_at` (TIMESTAMP, Nullable)
- `cancelled_at` (TIMESTAMP, Nullable)

### 8. `payments` (Cost-share records, 1:1 with Bookings)
- `payment_id` (PK, BIGINT, Auto-Increment)
- `booking_id` (FK → `bookings.booking_id`, UNIQUE, CASCADE updates/deletes)
- `amount` (DECIMAL(10, 2), NOT NULL)
- `settlement_method` (ENUM: `'upi', 'cash', 'in_app'`, NOT NULL)
- `payment_status` (ENUM: `'pending', 'settled', 'disputed'`, Default: `'pending'`)
- `settled_at` (TIMESTAMP, Nullable)
- `created_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP`)

### 9. `ratings` (Star feedback, CHECK 1-5, N:1 with Bookings/Users)
- `rating_id` (PK, BIGINT, Auto-Increment)
- `booking_id` (FK → `bookings.booking_id`, CASCADE updates/deletes)
- `rater_id` (FK → `users.user_id`, CASCADE updates/deletes)
- `rated_user_id` (FK → `users.user_id`, CASCADE updates/deletes)
- `stars` (INT, CHECK between 1 and 5, NOT NULL)
- `created_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP`)

### 10. `reviews` (Text feedback, N:1 with Bookings/Users)
- `review_id` (PK, BIGINT, Auto-Increment)
- `booking_id` (FK → `bookings.booking_id`, CASCADE updates/deletes)
- `reviewer_id` (FK → `users.user_id`, CASCADE updates/deletes)
- `reviewed_user_id` (FK → `users.user_id`, CASCADE updates/deletes)
- `review_text` (TEXT, NOT NULL)
- `is_moderated` (BOOLEAN, Default: `false`)
- `created_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP`)

### 11. `reports` (Behavior/Safety Reports - Immutable logic)
- `report_id` (PK, BIGINT, Auto-Increment)
- `reporter_id` (FK → `users.user_id`, CASCADE updates/deletes)
- `reported_user_id` (FK → `users.user_id`, CASCADE updates/deletes)
- `ride_id` (FK → `rides.ride_id`, Nullable, SET NULL deletes)
- `booking_id` (FK → `bookings.booking_id`, Nullable, SET NULL deletes)
- `category` (ENUM: `'unsafe_driving', 'misbehavior', 'fake_account', 'harassment', 'no_show', 'other'`, NOT NULL)
- `detail` (TEXT, NOT NULL)
- `urgency` (ENUM: `'standard', 'urgent'`, Default: `'standard'`)
- `status` (ENUM: `'received', 'under_review', 'resolved'`, Default: `'received'`)
- `resolved_by_admin_id` (FK → `users.user_id`, Nullable, SET NULL deletes)
- `resolution_note` (TEXT, Nullable)
- `created_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP`)
- `resolved_at` (TIMESTAMP, Nullable)

### 12. `notifications` (Notification Dispatch History)
- `notification_id` (PK, BIGINT, Auto-Increment)
- `user_id` (FK → `users.user_id`, CASCADE updates/deletes)
- `notification_type` (ENUM: `'ride_reminder', 'booking_status', 'rating_prompt', 'system_alert', 'safety_alert', 'other'`, NOT NULL)
- `title` (VARCHAR(255), NOT NULL)
- `body` (TEXT, NOT NULL)
- `is_read` (BOOLEAN, Default: `false`)
- `metadata` (JSON, Nullable)
- `created_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP`)

### 13. `emergency_alerts` (SOS Trigger Logs - Immutable logic)
- `alert_id` (PK, BIGINT, Auto-Increment)
- `user_id` (FK → `users.user_id`, CASCADE updates/deletes)
- `ride_id` (FK → `rides.ride_id`, Nullable, SET NULL deletes)
- `trigger_location` (POINT (SRID 4326), NOT NULL)
- `status` (ENUM: `'active', 'resolved', 'false_alarm'`, Default: `'active'`)
- `triggered_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP`)
- `resolved_at` (TIMESTAMP, Nullable)
- `admin_notes` (TEXT, Nullable)

### 14. `emergency_contacts` (Mandatory contacts, N:1 with Users)
- `contact_id` (PK, BIGINT, Auto-Increment)
- `user_id` (FK → `users.user_id`, CASCADE updates/deletes)
- `contact_name` (VARCHAR(255), NOT NULL)
- `contact_phone` (VARCHAR(50), NOT NULL)
- `is_verified` (BOOLEAN, Default: `false`)
- `created_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP`)

### 15. `verification_records` (Trust/ID status histories)
- `verification_id` (PK, BIGINT, Auto-Increment)
- `user_id` (FK → `users.user_id`, CASCADE updates/deletes)
- `verification_type` (ENUM: `'mobile', 'email', 'institutional', 'government_id'`, NOT NULL)
- `status` (ENUM: `'pending', 'verified', 'rejected', 'expired'`, Default: `'pending'`)
- `reference_value` (VARCHAR(255), NOT NULL)
- `reviewed_by_admin_id` (FK → `users.user_id`, Nullable, SET NULL deletes)
- `submitted_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP`)
- `verified_at` (TIMESTAMP, Nullable)
- `expires_at` (TIMESTAMP, Nullable)

### 16. `audit_logs` (System Immutable Action Trails)
- `log_id` (PK, BIGINT, Auto-Increment)
- `actor_user_id` (FK → `users.user_id`, Nullable, SET NULL deletes)
- `target_user_id` (FK → `users.user_id`, Nullable, SET NULL deletes)
- `action_type` (VARCHAR(100), NOT NULL)
- `action_detail` (JSON, NOT NULL)
- `created_at` (TIMESTAMP, Default: `CURRENT_TIMESTAMP`)

---

## 2. Table Indexing Strategy

We added indices corresponding to the required lookup structures:
1. `rides`: `(ride_date, status)`
2. `rides`: `(source_location)` (**SPATIAL INDEX** - SRID 4326)
3. `rides`: `(destination_location)` (**SPATIAL INDEX** - SRID 4326)
4. `ride_stops`: `(stop_location)` (**SPATIAL INDEX** - SRID 4326)
5. `ride_stops`: `(ride_id, sequence_order)` (composite unique index)
6. `bookings`: `(passenger_id, booking_status)`
7. `bookings`: `(ride_id, booking_status)`
8. `reports`: `(status, urgency)`
9. `notifications`: `(user_id, is_read, created_at)`
10. `verification_records`: `(user_id, verification_type, status)`
11. `emergency_alerts`: `(status, triggered_at)`
